PMKSA Caching, OKC, FT (802.11r) — Keying Hierarchy

Prereq: 4-way handshake familiarity (assumed known)
Socratic Questions
In FT-over-DS, the client sends an FT Action frame to AP1 (the current AP) addressed to AP2 (the target). AP1 relays it. What frame carries the PMK-R1 derivation request from AP2 to AP1 — and what key material does AP1 send back?
OKC (Opportunistic Key Caching) caches the PMK at the controller, not per-AP. If you have 3 APs and a client roams A→B→C→A, how many full 802.1X authentications does the client perform — and what's the PMKID derivation formula that makes the cache lookup work?

Apple Wireless Documents — roaming PDF set (T2)
80211-2024.pdf §13.4 — FT key hierarchy (PMK-R0, PMK-R1, PTK derivation)

Lab Exercise
FT Auth Frame Capture + Latency Delta
Create two SSIDs on AP1: one with 802.11r enabled, one without → iPhone roam between radios → capture FT Auth → Reassoc sequence → measure deauth-to-first-data-frame delta for FT vs full 4-way → record latency savings in ms.

Day 2 of 6

SAE — H2E (Hash-to-Element), PT Pre-Computation, Anti-Clogging

Prereq: SAE commit/confirm basics (assumed known)
Socratic Questions
Original SAE "Hunting-and-Pecking" (H2P) had a timing side-channel — the loop iteration count leaked information about the password. What does H2E (Hash-to-Element) specifically change to eliminate that side-channel?
The PT (Password Token) pre-computation in H2E is done once at provisioning and stored. If an AP supports both H2P and H2E clients simultaneously, what IE field advertises capability — and can a client be silently downgraded to H2P by a rogue AP?
Reading (T1 Corpus)

80211-2024.pdf §12.4 — SAE authentication, H2E, anti-clogging token
802.11be-2024.pdf — security addendum (SAE-PT)

Lab Exercise
SAE Commit/Confirm Frame Decode
Configure AP1 with WPA3-Personal (SAE-only) SSID → AP1 capture during iPhone join → Wireshark filter wlan.fc.type_subtype == 0x000b (Authentication) → decode SAE Commit: Finite Cyclic Group, Anti-Clogging Token (if present), Scalar, Element fields.

MFP / 802.11w / BIP — Protected Management Frames in Practice

Prereq: Day 2 (SAE); prior rogue NAV PCAP from May 4 session
Socratic Questions
MFP "Required" mode rejects any client that doesn't support 802.11w. If a rogue AP sends a spoofed Deauth (unprotected) to disconnect a client, what does the client with MFP Required do — and what is the SA Query procedure that follows?
BIP-CMAC (Broadcast Integrity Protocol) protects group-addressed management frames. Which specific frame types does it protect, and why can't it use per-unicast encryption keys?

OWE + OWE Transition Mode

Prereq: Day 2–3 (SAE, MFP); DHCP open network context
Socratic Questions
OWE Transition Mode runs two BSSIDs on the same AP: one open (legacy), one OWE. When an iPhone sees both, what determines which BSSID it associates to — and what specific IE field in the open BSSID's beacon points at the OWE BSSID?
OWE provides unauthenticated encryption. Why is it still meaningful for a coffee-shop captive portal — and what attack does it specifically prevent that WPA2-Open cannot?

802.1X EAP-TLS — Server Cert Chain, MAC Randomization vs RADIUS

Prereq: PEAP-MSCHAPv2 flow (assumed known); Phase 2 Day 6 (multicast/ARP)
Socratic Questions
If a MacBook sends a per-SSID random MAC (iOS 14+/macOS 12+ default) and your RADIUS policy enforces MAC-based authorization, what breaks first — accounting continuity, or authorization — and which RADIUS attribute carries the offending value?
EAP-TLS uses mutual certificate authentication. The client presents a cert, the server presents a cert. If the server cert chain is missing an intermediate CA, what happens in the EAP exchange — and where does it fail in the handshake?

Roaming Triggers — 11k Neighbor Report, BTM (11v), Apple Roaming Algorithm

Prereq: Day 1 (FT/OKC); Phase 2 Day 5 (power save, wake events)
Socratic Questions
Apple's roaming algorithm requires the candidate AP to have a better RSSI than the current AP by a defined hysteresis margin. If the controller sends a BTM Request with Disassociation Imminent = 1, what does the iPhone do — and under what condition does it ignore the request?
11k Neighbor Report gives the client a list of candidate APs without scanning every channel. If the neighbor list is stale (AP decommissioned), what does the client do — and how does the AP know to refresh the list?
